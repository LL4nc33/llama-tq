# RULER + LongBench Harness ‚Äî Skeleton

Spec: [`docs/plans/2026-04-26-ruler-harness-spec.md`](../../docs/plans/2026-04-26-ruler-harness-spec.md)

This directory will contain the long-context evaluation harness for the
llama-tq fork: synthetic RULER tasks (primary) and real LongBench tasks
(secondary), driven through the llama-server `/completion` endpoint.

**Status: skeleton only.** End-to-end wiring works against a locally-spun
llama-server, but scoring uses a placeholder substring metric. The real
RULER + LongBench scorers are vendored later ‚Äî see
[`vendor/VENDOR.md`](vendor/VENDOR.md) for target upstream SHAs.

## Dependencies

- Python 3.10+ (stdlib only ‚Äî `urllib`, `json`, `csv`, `argparse`)
- A reachable `llama-server` exposing `/completion`

No `pip install` required for the skeleton. Once RULER and LongBench are
vendored, additional deps (e.g. `transformers` for tokenizer-aware length
budgeting) will land via a `requirements.txt` next to this README.

## Running the Smoke

**Do not run against the live deploy on gpu00:8791.** Spin a local server
with the smallest test model first:

```bash
# Terminal A ‚Äî local llama-server with the smallest test model
./build/bin/llama-server -m /path/to/qwen3.5-0.8b-q8_0.gguf \
    --port 8080 --host 127.0.0.1 -c 8192

# Terminal B ‚Äî drive the smoke
cd bench/harness
./run_smoke.sh http://localhost:8080
```

Expected output: a `work/smoke/summary/smoke.csv` line of the form
`niah_single_3,10,<acc>` and the per-task accuracy printed to stderr.

The placeholder NIAH samples generated by `run_smoke.sh` are *not* the
real RULER tasks; they exist so the pred ‚Üí score plumbing can be
exercised before the vendor step. Treat the accuracy as a wiring signal
only, not a quality signal.

## Constraints

- `cache_prompt: false` always ‚Äî KV-cache reuse masks TurboQuant
  regressions. Do not optimise this away.
- Single-stream evals: `parallel: 1`, `cont_batching: false` for
  determinism on MoE models (Qwen3.6-A3B expert routing is order-sensitive).
- No parallel bench runs on gpu00 (see project memory rule).
